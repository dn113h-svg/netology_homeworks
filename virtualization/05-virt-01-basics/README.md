
# Ответы по заданию к занятию 1.  «Введение в виртуализацию»

## Задача 2

- высоконагруженная база данных MySql, критичная к отказу - лучше один физический сервер, чтобы субд не приходилось делиться ресурсами.
- различные web-приложения - паравиртуализация либо виртуализация уровня ОС в зависимости от критичности скорости развертывания;
- Windows-системы для использования бухгалтерским отделом - в зависимости от вычислительных мощностей и архитектуры организации. Можно обойтись одной машиной на Windows server. Если внутри организации есть мощный сервер и еще какие-либо сервисы, можно развернуть их на сервере при помощи паравиртуализации или виртуализации уровня ОС.
- системы, выполняющие высокопроизводительные расчёты на GPU - можно обойтись одним сервером. Паравиртуализация будет отнимать ресурсы, а виртуализация уровня ОС не требуется, так как речь об одном сервисе. Если и есть сеть датацентров, выполняющих рассчеты на GPU, то контроль нагрузки на них лежит вне поставленной задачи.

## Задача 3

1. Упомянутым требованиям, судя по доке, соответствует VMware.
2. Описаному сценарию более всего соответствует Xen.
3. Если инфраструктура полностью на Windows, то и для виртуализации оптимальнее всего использовать Hyper-V.
4. Оптимальнее всего KVM.
## Задача 4

Основной проблемой при использовании гетерогенной среды является то, что каждый инструмент виртуализации будет требовать отдельного навыка для работы с ним, что поывсит нагрузку и риск ошибок со стороны инженера. По возможности я бы выбирал наиболее универсальный инструмент, в противном случае возможно пришлось бы создать отдельный инструмент, который по возможности позволил бы абстрагироваться от интерфейса управления каждой из сред.
